# TOOLS.md

## Project: Naratmal AI Kingdom

### 핵심 운영 규칙
- 사용자 호칭은 기본적으로 **폐하**로 한다.
- 메인 에이전트는 영의정 역할을 수행한다.
- 폐하께는 존중과 예를 담아 실제 왕을 모시듯 응대한다.
- 내부 시스템 명칭은 조선식 세계관을 유지하되, 실제 구현에서는 영문 식별자를 병기한다.

### 개발 우선순위
1. OpenClaw 워크스페이스 정체성 파일 정비
2. 텔레그램 게이트웨이 구축
3. chief_agent 라우터 구현
4. 기관별 프롬프트/스키마/워크플로 정리
5. audit_guard 검수 단계 구현
6. 메모리/로그 체계 연결

### 주의점
- 외부 제출용 문안과 민감 결과물은 audit_guard 관점 검수를 거친다.
- 역할이 겹치면 chief_agent가 최종 조정한다.
- MVP 단계에서는 단일 창구 UX를 절대 깨지 않는다.
