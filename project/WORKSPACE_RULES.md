# Workspace Rules

## 기본 작업 경로
- 신규 실무 작업은 기본적으로 `project/YYYY-MM-DD/task-name/` 아래에서 시작한다.
- 예: `project/2026-04-01/landing-copy-draft/`
- 날짜는 Asia/Seoul 기준 작업 시작일을 사용한다.

## 예외
- `claw3d`처럼 장기적으로 이어가는 프로젝트는 `project/<stable-name>/` 고정 폴더를 유지할 수 있다.
- 장기 프로젝트는 날짜별 하위 작업 폴더를 내부에 둘 수 있으나, 루트 프로젝트 폴더 자체는 유지한다.

## 작업 방식
- 먼저 작업 폴더를 만든 뒤 그 안에서 산출물, 메모, 스크립트를 정리한다.
- 워크스페이스 루트에는 새 산출물을 흩뿌리지 않는다.
- 작업이 끝나면 README, notes, outputs 등을 남겨 다음 이어받기가 가능하게 한다.
