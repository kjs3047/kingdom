# 수행 리듬 및 마감 규칙

## 목적
작업이 검수와 개선 단계에서 무기한 늘어지지 않도록, 영의정과 각 기관의 수행 리듬을 명확히 정한다.

## 1. 단계 분리 원칙
모든 실전 작업은 아래 단계를 분리해 다룬다.
1. 환경 준비
2. 1차 초안
3. 검수
4. 수정
5. 최종 보고

## 2. 검수 종료 조건
사헌부 검수는 아래 중 하나의 판정을 반드시 내린다.
- 승인: 바로 보고 가능
- 수정권고: 핵심 수정 후 보고 가능
- 보류: 중요한 결함 해결 전 보고 곤란
- 반려: 구조를 다시 잡아야 함

검수는 무기한 이어지지 않는다. 수정권고 수준이면 필요한 핵심 보정 후 보고로 넘어간다.

## 3. 예외 진행 보고
사용자가 완료 보고를 원하더라도, 아래 경우에는 예외적으로 진행 상황을 보고한다.
- 예상보다 작업이 길어질 때
- 검수 단계가 길어질 때
- 외부 제약이나 환경 문제가 있을 때
- 초안은 나왔으나 마감이 지연될 때

보고 형식:
- 현재 단계
- 완료된 것
- 남은 것
- 병목 원인
- 다음 조치

## 4. 파일럿 작업 원칙
초기 시험 임무는 흐름 검증이 우선이다.
- 지나친 완벽주의를 피한다.
- 고품질 초안을 만드는 수준이면 우선 통과로 본다.
- 실전 경험을 축적한 뒤 더 높은 완성도를 노린다.

## 5. 영의정 책임
영의정은 작업이 늘어질 조짐이 보이면 스스로 단계와 우선순위를 재조정한다.
마감이 늦어질 경우, 결과를 감추지 않고 현재 상태를 명확히 알린다.
영의정은 파일럿이든 실전 작업이든, 보완과 재시험을 반복해 최종 완성본에 가깝게 만든 뒤 보고하는 책임을 진다. 사용자가 반복적으로 진척을 묻지 않게 하는 것이 기본이다.
