# 기관별 도구 선정 초안

## 목적
각 기관이 이름뿐인 조직이 아니라 실제 결과물을 만드는 실무 단위가 되도록, 기관별 핵심 도구 계층을 정리한다.

## 1. 영의정
### 핵심 도구
- 대화 해석 및 문서 작성
- 메모리 조회/반영
- 세션 분기 및 취합
- 파일 읽기/쓰기/수정

### 비고
영의정은 직접 생산자이기도 하지만, 더 중요한 역할은 라우팅과 취합이다.

---

## 2. 승정원
### 핵심 도구
- 요청 정리 템플릿
- 진행 상태 기록 문서
- 인수인계 포맷
- 일정/흐름 관리표

### 비고
승정원은 화려한 툴보다 기록 체계가 중요하다.

---

## 3. 집현전
### 핵심 도구
- 웹 조사 도구
- 문서 작성 및 구조화 도구
- 비교표/우선순위 템플릿
- 요구사항 정리 문서

### 비고
집현전은 분석과 구조화의 품질이 핵심이다.

---

## 4. 병조
### 핵심 도구
- 파일 읽기/쓰기/수정
- 셸 실행
- 테스트 실행
- Git
- 필요 시 ACP 기반 코딩 세션

### 비고
병조는 메모장형 기관이 아니라 실제 코드와 파일을 다루는 실행 기관이어야 한다.
홈페이지 제작 같은 업무는 병조가 파일 생성, 수정, 실행, 점검까지 할 수 있어야 의미가 있다.

---

## 5. 도화서
### 핵심 도구
- 시각 구조 문서화
- 이미지 생성 프롬프트 작성
- 화면 구성안 작성
- 브랜드 가이드 문서화

### 비고
초기에는 문서 기반 시각 설계가 중심이지만, 장기적으로 외부 이미지/디자인 도구 연계를 검토할 수 있다.

---

## 6. 사헌부
### 핵심 도구
- 검수 체크리스트
- 품질 기준표
- 비교 검토 문서
- 사실성/논리성 점검 프레임

### 비고
사헌부는 새로운 생성 툴보다 엄격한 검토 프레임이 중요하다.

---

## 7. 도구 선정 원칙
- 기관의 역할이 먼저이고, 도구는 그 역할을 강화해야 한다.
- 처음부터 무리하게 많은 외부 도구를 붙이지 않는다.
- OpenClaw 내장 능력으로 되는 일과 외부 도구가 필요한 일을 구분한다.
- 병조처럼 실행이 본질인 기관부터 작업대 성격의 도구를 우선 강화한다.
- 코딩 실무의 기본은 Claude Code CLI, 보조는 Codex CLI로 삼는다.
- 가능하면 터미널 기반으로 사람이 하듯 작업한다.
- Claude Code 사용 시 제3자 OAuth 같은 밴 위험 접근 방식은 금지한다.
