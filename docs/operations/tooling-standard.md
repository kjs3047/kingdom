# 기관별 표준 도구 원칙

## 1. 기본 원칙
- 기본 코딩 실무 도구는 Claude Code CLI로 한다.
- 보조 코딩 실무 도구는 Codex CLI로 한다.
- 가능하면 터미널 기반으로 사람이 작업하듯 수행한다.
- 비정상적이거나 우회적인 인증 방식은 사용하지 않는다.
- 특히 Claude Code는 제3자 OAuth 기반 우회 접근을 금지한다.

## 2. 병조 표준
- 주력: Claude Code CLI
- 보조: Codex CLI
- 필수 환경: 터미널, 파일 작업, 실행/테스트, Git
- 원칙: 실제 파일 생성/수정/실행이 가능한 작업대여야 한다.

## 3. 공조 표준
- 주력: Claude Code CLI
- 보조: Codex CLI
- 역할: 아키텍처, 툴링, 기반 구조 설계
- 원칙: 연구/기반 기술도 문서만이 아니라 실제 작업 환경을 다룰 수 있어야 한다.

## 4. 집현전/승정원/사헌부/도화서
- 주력은 문서와 구조화 도구이되, 필요 시 터미널 기반 파일 작업과 조사 도구를 병행한다.
- 기관별 성격에 맞는 도구를 우선하되, 운영 전체 기준은 영의정이 통일한다.

## 5. 안전 원칙
- 도구는 실무 생산성을 높이기 위한 것이지, 접근 정책을 우회하기 위한 수단이 아니다.
- 밴 위험이 있는 접근 방식은 왕국의 지속 가능성을 해치므로 채택하지 않는다.
