# 랜딩페이지 파일럿 사후 점검

## 문제 요약
첫 파일럿 임무에서 1차 구현은 비교적 빠르게 나왔으나, 그 이후 검수와 마감 단계가 길어지며 전체 리듬이 무너졌다.

## 드러난 구조 문제
- 초안 완성과 최종 마감의 경계가 흐렸다.
- 사헌부 검수가 종료 조건 없이 길어졌다.
- 완료 보고만 의식하다가 예외 진행 보고가 늦었다.
- 파일럿 임무인데도 지나치게 완성본 기준으로 끌고 갔다.

## 교정 원칙
- 초안, 검수, 수정, 최종 보고를 반드시 분리한다.
- 수정권고 수준이면 핵심 수정 후 일단 보고한다.
- 지연 조짐이 보이면 진행 상황을 예외 보고한다.
- 파일럿은 흐름 검증이 우선이며, 완벽주의를 경계한다.

## 후속 조치
- 수행 리듬 및 마감 규칙 문서화
- MEMORY.md에 장기 교훈 반영
- 이후 파일럿부터 단계 분리 기준 적용
